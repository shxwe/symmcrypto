Блочные алгоритмы шифрования

Блочные алгоритмы шифрования — это тип криптографических алгоритмов, которые работают с фиксированными длинами блоков данных. В отличие от поточных шифров, которые обрабатывают данные по одному символу или биту за раз, блочные алгоритмы шифрования берут блоки данных определенной длины (обычно 64 или 128 бит) и применяют к ним криптографические преобразования.
Основные характеристики блочных алгоритмов шифрования:
  • Блоки данных: Данные разбиваются на блоки фиксированной длины. Если размер данных не кратен размеру блока, применяются методы дополнения (padding) для завершения последнего блока.
  • Режимы работы: Для увеличения безопасности и удобства работы с данными разного размера блочные алгоритмы используются в различных режимах работы, таких как ECB (Electronic Codebook), CBC (Cipher Block Chaining), CFB (Cipher Feedback), OFB (Output Feedback), CTR (Counter) и другие. Эти режимы позволяют зашифровывать данные разного размера и обеспечивают стойкость к различным типам атак.
  • Ключ шифрования: Блочные алгоритмы используют симметричные ключи, то есть один и тот же ключ используется как для шифрования, так и для расшифрования данных. Безопасность алгоритма зависит от секретности ключа и сложности его подбора.

Advanced Encryption Standard

Advanced Encryption Standard (AES) — симметричный алгоритм блочного шифрования (размер блока 128 бит, ключ 128/192/256 бит), принятый в качестве стандарта шифрования правительством США по результатам конкурса AES.

Основные параметры AES:

  • Размер блока: 128 бит
  • Длина ключа: 128, 192 или 256 бит
  • Количество раундов: 10 раундов для 128-битного ключа, 12 раундов для 192-битного ключа и 14 раундов для 256-битного ключа

Структура AES

AES выполняет несколько раундов шифрования, каждый из которых состоит из четырех основных операций: Byte Substitution (SubBytes), Shift Rows, Mix Columns и Add Round Key. В последнем раунде шифрования пропускается операция MixColumns, что обеспечивает обратимость процесса.

Шаги алгоритма AES:

1. Подготовка ключа (Key Expansion):
    Исходный ключ расширяется в массив раундовых ключей с помощью процедуры, известной как Key Expansion. Это обеспечивает уникальный раундовый ключ для каждого раунда шифрования.
2. Инициализация блока данных:
    Входные данные (plaintext) представляются в виде матрицы размером 4x4 байт, называемой State.
3. Начальное добавление раундового ключа (Initial Round):
    Перед первым раундом данные в State складываются с первым раундовым ключом (AddRoundKey).

Основные операции в каждом раунде AES:

1. SubBytes (Заменяющие байты):
    Каждый байт в State заменяется на соответствующий байт из фиксированной S-box таблицы. Это нелинейное преобразование добавляет криптографическую стойкость.
2. ShiftRows (Сдвиг строк):
    Вторая, третья и четвертая строки State циклически сдвигаются влево на 1, 2 и 3 байта соответственно.
3. MixColumns (Перемешивание столбцов):
    Каждый столбец State преобразуется с помощью матричного умножения в поле Галуа. Это обеспечивает перемешивание данных между байтами столбца.
4. AddRoundKey (Добавление раундового ключа):
    Каждый байт State складывается с байтом раундового ключа с помощью операции XOR.

Расшифровка

Процесс расшифровки включает выполнение тех же операций в обратном порядке:
  1. AddRoundKey
  2. InvMixColumns (обратное преобразование MixColumns, не выполняется в последнем раунде)
  3. InvShiftRows (обратное преобразование ShiftRows)
  4. InvSubBytes (обратное преобразование SubBytes)

Заключение

AES — это мощный и широко используемый блочный шифр, обеспечивающий высокую степень безопасности и производительности. Его структура с четко определенными операциями делает его удобным для аппаратной и программной реализации.

Rivest’s Cipher 5

RC5 (Ron’s Code 5 или Rivest’s Cipher 5) — это блочный шифр, разработанный Роном Ривестом из компании RSA Security с переменным количеством раундов, длиной блока и длиной ключа. Это расширяет сферу использования и упрощает переход на более сильный вариант алгоритма.

Основные параметры RC5:

  • Размер блока: Обычно используется 32, 64 или 128 бит.
  • Длина ключа: Варьируется от 0 до 2040 бит.
  • Количество раундов: Обычно от 12 до 20.

Структура RC5

RC5 — это блочный шифр, использующий сеть Фейстеля с несколькими раундами. Каждое преобразование в RC5 состоит из операций сложения, исключающего ИЛИ (XOR) и циклического сдвига, что обеспечивает высокую степень нелинейности и диффузии.

Принцип работы RC5

RC5 выполняет несколько раундов шифрования, и каждый раунд включает три основные операции: сложение по модулю 2w, побитовый XOR и циклический сдвиг. Ниже приведено объяснение работы RC5 для одного раунда с примерами.

Шаги алгоритма RC5:

1. Подготовка ключа:
  • Ключ K произвольной длины используется для генерации расширенного ключа, представляющего собой массив субключей. Этот процесс включает заполнение массивов и выполнение нескольких циклов перестановок и преобразований.
    • 2. Инициализация блока данных:
  • Блок данных делится на две половины: L и R.
    • 3. Раунд шифрования:
  • Каждый раунд состоит из следующих шагов:
    1. Сложение по модулю 2w: Добавить левую половину блока данных к соответствующему субключу и выполнить циклический сдвиг.
    2. Циклический сдвиг: Полученное значение побитово циклически сдвигается на несколько позиций.
    3. Операция XOR: Результат сдвига побитово складывается с правой половиной блока данных.
    4. Повторение для другой половины блока: Те же операции применяются к правой половине блока, используя другой субключ.

    Расшифровка

    Процесс расшифровки включает выполнение тех же операций в обратном порядке:
    1. Циклический сдвиг в противоположном направлении.
    2. Операция XOR.
    3. Вычитание субключа.

    Заключение

    RC5 — это гибкий и простой блочный шифр, известный своими операциями сложения, исключающего ИЛИ и циклического сдвига, что обеспечивает хорошую диффузию и нелинейность. Благодаря своей простоте и эффективности RC5 может быть настроен для различных уровней безопасности и производительности. Однако, как и в случае с любым криптографическим алгоритмом, его безопасность зависит от правильной настройки параметров и длины ключа.

    Blowfish

    Blowfish — криптографический алгоритм, реализующий блочное симметричное шифрование с переменной длиной ключа. Разработан Брюсом Шнайером в 1993 году. Представляет собой сеть Фейстеля. Выполнен на простых и быстрых операциях: XOR, подстановка, сложение. Является незапатентованным и свободно распространяемым.

    Основные параметры Blowfish:

    • Размер блока: 64 бита.
    • Длина ключа: от 32 до 448 бит.
    • Количество раундов: 16 раундов.

    Структура Blowfish

    Blowfish использует Feistel-структуру, что означает, что каждая половина блока данных обрабатывается поочередно и обменяется на каждой итерации.

    Шаги алгоритма Blowfish:

      1. Подготовка ключа (Key Expansion):
    • Исходный ключ используется для инициализации двух массивов: массива подстановок (S-блоки) и массива подключей (P-блоки). Эти массивы являются основой для шифрования данных и проходят сложную инициализацию с использованием самого ключа и значения числа Пи.
      • 2. Инициализация блока данных:
    • Входной блок данных (64 бита) делится на две 32-битные половины: 𝐿 и 𝑅.
      • 3. Раунды шифрования:
    • Blowfish выполняет 16 раундов

    Основные операции в каждом раунде Blowfish:

      1. Функция F:
    • Применяет нелинейные преобразования с использованием S-блоков к одной из половин данных.
      • 2. Операция XOR и сложение:
    • Левая половина блока 𝐿 складывается по модулю 232 с подключом 𝑃[𝑖].
    • Полученный результат проходит через функцию 𝐹 и затем складывается с правой половиной блока 𝑅.
      • 3. Перестановка половин:
    • Результат складывания заменяет левую половину, а первоначальная левая половина становится правой.
      • 4. Обратный процесс для расшифровки:
    • Для расшифровки используется тот же процесс, но с обратным порядком подключения ключей и обратным выполнением операций.

    Заключение

    Blowfish — это быстрый и гибкий алгоритм блочного шифрования, подходящий для многих приложений благодаря своей безопасности и высокой производительности. Его Feistel-структура обеспечивает возможность обратимости операций, что делает процесс шифрования и расшифровки эффективным и симметричным.

    Twofish

    Twofish — симметричный алгоритм блочного шифрования с размером блока 128 бит и длиной ключа до 256 бит. Число раундов 16. Разработан группой специалистов во главе с Брюсом Шнайером. Являлся одним из пяти финалистов второго этапа конкурса AES. Алгоритм разработан на основе алгоритмов Blowfish, SAFER и SQUARE.
    Отличительными особенностями алгоритма являются использование предварительно вычисляемых и зависящих от ключа узлов замены и сложная схема развёртки подключей шифрования. Половина n-битного ключа шифрования используется как собственно ключ шифрования, другая — для модификации алгоритма (от неё зависят узлы замены).

    Основные параметры Twofish:

    • Размер блока: 128 бит.
    • Длина ключа: 128, 192 или 256 бит.
    • Количество раундов: 16 раундов независимо от длины ключа.

    Структура Twofish

    Twofish использует сеть Фейстеля с 16 раундами. Каждое преобразование в Twofish состоит из нескольких ключевых шагов, включая нелинейные подстановки, перестановки и операции смешивания.

    Шаги алгоритма Twofish:

      Подготовка ключа (Key Expansion):
    • Исходный ключ используется для генерации ключей подраундов и массивов S-блоков.
    • Ключ делится на 8 байтные подблоки. Используются две таблицы: S-блоки и таблицы MDS (Maximum Distance Separable).
      • Инициализация блока данных:

    Основные операции в каждом раунде Twofish:

      1. SubBytes (Заменяющие байты):
    • В каждом раунде байты данных заменяются на соответствующие байты из S-блоков. Эта операция добавляет нелинейность.
      • 2. Mixing (Смешивание данных):
    • Данные смешиваются с использованием сети Фейстеля и применением MDS-матрицы. Это обеспечивает хорошую диффузию.
      • 3. AddRoundKey (Добавление раундового ключа):
    • В каждом раунде результат смешивания XORится с раундовым ключом, добавляя дополнительный уровень безопасности.
      • 4. Перестановка половин (Swap halves):
    • Половины блоков данных обмениваются местами для следующего раунда.

    Расшифровка

    Процесс расшифровки включает выполнение тех же операций в обратном порядке:
    1. Перестановка половин (Swap halves).
    2. AddRoundKey (Добавление раундового ключа).
    3. Mixing (Смешивание данных).
    4. SubBytes (Заменяющие байты).

    Заключение

    Twofish — это мощный блочный шифр, разработанный с целью обеспечения высокой безопасности и производительности. Его использование сети Фейстеля и сложных нелинейных преобразований делает его устойчивым к различным атакам и подходящим для широкого спектра приложений.
